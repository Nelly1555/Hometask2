## ***Неделя 2***



Создан зоопарк, для животных реализована иерархия классов:

```Animals```:
* ```Carnivorous```

    1. ```Duck```
    2. ```Fish```
    3. ```Wolf```

* ```Herbivore```:
    1. ```Goat```
    2. ```Butterfly```
    3. ```Beaver```


Создана аналогичная архитектура для классов еды:
* ```Food```:

    1. ```Grass```
    2. ```Meat```

Созданы интерфейсы, определяющие поведение животных: 

* ```Run```
* ```Swim```
* ```Fly ```
* ```Voice```

Классы животных реализуют интерфейсы, соответствующие данным конкретным животным. 

В классе Animals должен реализован метод ```eat (Food food)```, который принимает в качестве
аргумента еду. 

Если переданная еда не подходит животному (хищники не едят траву, травоядные не едят мясо), то на экран выводится соответствующее предупреждение. Проверка на соответствие типа еды выполняется
через ```instanceof```.

Создан класс рабочего зоопарка (```Worker```) с методами:
- ```feed(…)``` – покормить любое животное любой едой. У метода должно быть два
параметра – животное и еда.
- ```getVoice(…)``` – заставить животное подать голос (вывести на экран то, что говорит
конкретное животное). У метода один параметр – существо, которое издает звук. Метод принимает в качестве аргумента только тех животных, которые могут издавать звуки.

В корневом каталоге программы реализован класс ```Zoo``` c точкой входа в программу.
Созданы объекты классов:
* животных, 
* еды, 
* работника зоопарка. 

У объекта рабочего вызваны все его методы с разными комбинациями параметров (например попытаться покормить хищника и мясом и травой).

Так же создан пруд – массив с животными которые умеют плавать. В цикле вызван метод ```swim()``` у каждого животного из массива.

Животные имеют шкалу сытости, которая пополняется при кормлении. Методы выводят на экран различные действия, при выполнении которых единицы сытости уменьшаются.
